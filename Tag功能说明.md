# Tag功能实现说明

## 功能概述

Tag功能根据用户与AI的交互，自动生成搜索量最高的10个热门问题标签，用户点击Tag可以快速获取答案。

## 实现原理

### 1. 问题记录
- 用户每次提问时，系统自动记录问题内容
- 使用MD5哈希值去重，相同问题会累加提问次数
- 记录最后提问时间，用于时间衰减计算

### 2. Tag生成算法
- **数据来源**：最近7天的用户提问数据
- **排序规则**：按提问次数降序排列
- **得分计算**：`得分 = 提问次数 × 时间衰减系数`
  - 时间衰减系数：距离现在越近，系数越大（最大1.0，最小0.5）
- **Top N选择**：选择得分最高的N个问题作为Tag

### 3. Tag答案生成
- 为每个Tag调用AI生成标准答案
- 答案存储在数据库中，点击Tag时直接返回（秒级响应）
- 支持后续优化：可以人工编辑Tag答案

### 4. 定时任务
- 每天凌晨2点自动生成Top 10 Tag
- 也可以手动触发生成（管理员接口）

## 数据库表结构

### tb_customer_service_question（问题统计表）
- 记录用户提问的问题
- 统计提问次数、用户数、最后提问时间

### tb_customer_service_tag（Tag表）
- 存储生成的Tag信息
- 包含Tag名称、描述、答案、得分等

### tb_customer_service_tag_question（关联表）
- Tag与问题的关联关系（预留，用于后续扩展）

## API接口

### 1. 获取Top N Tag
```
GET /api/tag/top?topN=10
Response: [Tag列表]
```

### 2. 点击Tag获取答案
```
POST /api/tag/{tagId}/click
Response: { answer, tagId }
```

### 3. 手动生成Tag（管理员）
```
POST /api/tag/generate?topN=10
```

## 前端功能

### Tag显示
- 在客服页面顶部显示Top 10 Tag
- 热门Tag（点击次数>10）显示星标
- Tag支持点击，点击后快速返回答案

### 交互流程
1. 用户点击Tag
2. 系统显示用户消息（Tag名称）
3. 调用API获取Tag答案
4. 显示AI回复（从缓存返回，秒级响应）

## 使用说明

### 1. 数据库配置
执行 `schema.sql` 中的Tag相关表结构

### 2. 自动生成
- 系统每天凌晨2点自动生成Tag
- 首次使用需要等待有足够的问题数据

### 3. 手动生成
可以通过API手动触发Tag生成：
```bash
POST /api/tag/generate?topN=10
```

## 优化建议

### 1. 问题聚类
当前实现是简化版，可以进一步优化：
- 使用文本相似度算法对相似问题进行聚类
- 将相似问题归为一个Tag

### 2. 个性化Tag
- 根据用户历史行为推荐个性化Tag
- 新用户显示通用Tag，老用户显示个性化Tag

### 3. Tag答案优化
- 支持管理员手动编辑Tag答案
- 根据用户反馈优化答案内容

### 4. 实时更新
- 当新问题达到阈值时，立即更新Tag
- 支持增量更新，而不是全量重新生成

## 注意事项

1. **数据积累**：Tag生成需要足够的问题数据，建议至少运行几天后再生成
2. **API成本**：生成Tag时会调用AI API，注意控制成本
3. **答案缓存**：Tag答案已缓存，点击时不会再次调用AI API
4. **定时任务**：确保定时任务正常执行，可以查看日志确认
